# 安室智能 自动化运维平台 - 分阶段落地清单

## 1. 项目背景
- 这是一个自动化运维网站。
- 当前已有前后端基础框架（FastAPI + Vue3）。
- 已有独立 Python 脚本：
  - `D:\network-automation\huawei_audit.py`
  - `D:\network-automation\huawei_ntp_config.py`
- 网站负责触发脚本、展示结果。

## 2. 当前目标
- 完成设备管理模块。
- 完成首页基础统计。
- 完成任务记录基础能力。

## 3. 技术约束
- 保持现有项目结构。
- 以小改动为主，不重写已有脚本逻辑。
- 后端通过调用现有 Python 脚本执行任务。
- 风格支持：跟随系统 / 浅色 / 深色。

## 4. 需要交付内容
- 后端接口。
- 数据库模型。
- 前端页面与路由。
- 基础交互。
- 必要注释。
- 本地运行说明。

## 5. 明确边界
- 暂不做复杂权限系统。
- 暂不做消息队列。
- 暂不做分布式任务调度。
- 当前目标是单机可用版本。

## 6. 三轮实施顺序
### 第一轮：设备管理模块
- 设备列表、搜索、增删改。
- 前后端联通并写入 SQLite。

### 第二轮：巡检任务执行模块
- 前端触发巡检。
- 后端调用 `huawei_audit.py`。
- 写入 `tasks`、`task_results`，并展示结果。

### 第三轮：NTP 批量配置模块
- 前端填写 NTP 参数并执行。
- 后端调用 `huawei_ntp_config.py`。
- 写入任务记录并展示结果。
